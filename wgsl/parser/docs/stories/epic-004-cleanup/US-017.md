## 🔧 Usage

### Gradle

```kotlin
// build.gradle.kts
dependencies {
    implementation("com.github.yourorg:webgpu-ktypes:wgsl-parser:VERSION")
}
```

### Basic Parsing

```kotlin
import io.ygdrasil.wgsl.parser.Parser
import io.ygdrasil.wgsl.parser.lexer.Lexer

val source = """
    [[stage(vertex)]]
    fn main() -> [[builtin(position)]] vec4<f32> {
        return vec4<f32>(0.0, 0.0, 0.0, 1.0);
    }
""".trimIndent()

val parser = Parser(Lexer(source))
val translationUnit = parser.parse()

if (parser.errors.isNotEmpty()) {
    parser.errors.forEach { println("Error: $it") }
} else {
    // Success! Work with the AST
    println("Parsed successfully!")
}
```

### Working with Enums

```kotlin
val source = """
    enum Color { RED, GREEN, BLUE = 2 }
    
    [[stage(fragment)]]
    fn main() {
        let c: Color = Color.RED;
        let mode = AddressMode.clamp_to_edge;
    }
""".trimIndent()

val parser = Parser(Lexer(source))
val unit = parser.parse()
```

### Type Resolution

```kotlin
import io.ygdrasil.wgsl.parser.TypeResolver

val resolver = TypeResolver(translationUnit)
val type = resolver.resolve(expression)
```

# Enum Support in WGSL Parser

*Added in: EPIC-001*  
*Status: ✅ Complete*  
*Specification: WGSL Section 6.3*

---

## Overview

The WGSL parser now provides complete support for enumeration types, including both user-defined enums and predeclared enumerants as defined in the WGSL specification.

---

## User-Defined Enums

### Syntax

```wgsl
// Basic enum
enum Color {
    RED,
    GREEN,
    BLUE
}

// Enum with explicit values
enum Status {
    ACTIVE = 1,
    INACTIVE = 0,
    PENDING = 2
}

// Enum with mixed explicit and implicit values
enum Priority {
    LOW,       // 0
    MEDIUM,    // 1
    HIGH = 3,  // explicit
    URGENT     // 4 (implicit, follows HIGH)
}

// Enum with attributes
enum TextureType {
    [[id(0)]] DIFFUSE,
    [[id(1)]] NORMAL,
    [[id(2)]] SPECULAR
}
```

### Features

✅ **Enum Declaration**: Full support for `enum` keyword and declaration syntax  
✅ **Enum Members**: Members with and without explicit values  
✅ **Auto-assignment**: Automatic value assignment for unspecified members  
✅ **Attributes**: Support for attributes on enum declarations and members  
✅ **Type System**: Enums are first-class types in the type system  
✅ **Value Access**: Access enum values using `EnumName.MemberName` syntax  

### Usage Examples

```wgsl
// Variable with enum type
let color: Color = Color.RED;

// Function parameter with enum type
fn process(status: Status) {
    switch (status) {
        case Status.ACTIVE: { /* ... */ }
        case Status.INACTIVE: { /* ... */ }
    }
}

// Enum in struct
struct Material {
    textureType: TextureType,
    color: Color,
}

// Default parameter value
fn createMaterial(type: TextureType = TextureType.DIFFUSE) {
    // ...
}
```

---

## Predeclared Enumerants

### Overview

Predeclared enumerants are special enumeration values defined by the WGSL specification that do not require explicit declaration in user code.

### Supported Categories

#### AddressMode
Used for texture coordinate addressing.

| Value | Description |
|-------|-------------|
| `clamp_to_edge` | Clamps coordinates to [0, 1] |
| `repeat` | Repeats the texture |
| `mirror_repeat` | Mirrors the texture |

#### FilterMode
Used for texture filtering.

| Value | Description |
|-------|-------------|
| `nearest` | Nearest-neighbor filtering |
| `linear` | Bilinear filtering |

#### MipmapFilterMode
Used for mipmap filtering.

| Value | Description |
|-------|-------------|
| `nearest` | Nearest mipmap level |
| `linear` | Linear interpolation between mipmap levels |

#### InterpolationType
Used for interpolation type in stage I/O.

| Value | Description |
|-------|-------------|
| `perspective` | Perspective-correct interpolation |
| `linear` | Linear interpolation |
| `flat` | Flat (no interpolation) |

#### InterpolationSampling
Used for interpolation sampling in stage I/O.

| Value | Description |
|-------|-------------|
| `sample` | Interpolation at sample rate |
| `vertex` | Interpolation at vertex rate |
| `centroid` | Centroid sampling |

### Usage Examples

```wgsl
// Using AddressMode
[[group(0), binding(0)]]
var texture: texture_2d<f32>;

[[group(0), binding(1)]]
var sampler: sampler;

[[stage(fragment)]]
fn main() {
    let addressMode = AddressMode.clamp_to_edge;
    let texColor = textureSample(texture, sampler, coord, addressMode);
}

// Using FilterMode
[[stage(fragment)]]
fn main() {
    let filterMode = FilterMode.linear;
    // Use in sampling...
}

// Multiple enumerants
struct SamplerConfig {
    addressMode: AddressMode,
    filterMode: FilterMode,
    mipmapFilterMode: MipmapFilterMode,
}

// In function parameters
fn sampleTexture(
    texture: texture_2d<f32>,
    sampler: sampler,
    addressMode: AddressMode = AddressMode.repeat,
    filterMode: FilterMode = FilterMode.linear
) -> vec4<f32> {
    return textureSample(texture, sampler, coord, addressMode, filterMode);
}
```

## Implementation Details

### AST Representation

```kotlin
// User-defined enum
data class EnumDecl(
    val name: String,
    val members: List<EnumMember>,
    val attributes: List<Attribute>,
    val span: Span
) : GlobalDecl

data class EnumMember(
    val name: String,
    val value: Expression?,  // null for auto-assigned
    val attributes: List<Attribute>,
    val span: Span
)

// Predeclared enumerant
data class PredeclaredEnumerant(
    val category: String,
    val name: String,
    val value: Expression?,  // symbolic value
    val span: Span
)

// Expression reference
sealed class EnumValueExpr : Expression() {
    abstract val enumValue: EnumValue
    abstract fun getEnumTypeName(): String
}

data class EnumMemberExpr(
    val enumDecl: EnumDecl,
    val memberName: String,
    override val span: Span
) : EnumValueExpr()

data class PredeclaredEnumerantExpr(
    val category: String,
    val enumerant: PredeclaredEnumerant,
    override val span: Span
) : EnumValueExpr()
```

    fun isValidQualifiedName(qualifiedName: String): Boolean
    fun getAllEnumerants(): List<PredeclaredEnumerant>
}
```

# Abstract Numeric Types Support

*Added in: EPIC-002*  
*Status: ✅ Complete*  
*Specification: WGSL Section 6.2.1*

---

## Overview

Abstract numeric types (`abstract int` and `abstract float`) are generic numeric types that can represent any integer or floating-point type, respectively. They are used to provide flexibility in shader interfaces without committing to a specific concrete type.

---

## Syntax

```wgsl
// Abstract int - can represent any integer type
let x: abstract int;

// Abstract float - can represent any floating-point type
let y: abstract float;

// In function parameters
fn process(value: abstract int) {}

// In structs
struct GenericBuffer {
    size: abstract int,
    scale: abstract float,
}
```

---

## Features

✅ **Parsing**: Full parsing support for `abstract int` and `abstract float`  
✅ **Type System**: Recognized as distinct type categories  
✅ **AST**: Dedicated AST nodes for abstract types  
✅ **Type Resolution**: Integrated into the type resolver

## Validation Rules

The WGSL specification defines these rules for abstract types:

1. **Abstract int** can be used anywhere a signed integer type is expected
2. **Abstract float** can be used anywhere a floating-point type is expected
3. They cannot be instantiated directly (no literal has abstract type)
4. They can be used as:
   - Function parameter types
   - Function return types
   - Struct member types
   - Global variable types
   - Array element types
   - Pointer element types

### ❌ Invalid Usage

```wgsl

// CORRECT: The literal's type is compatible with abstract int
let x: abstract int = i32(42);  // OK - explicit constructor

// ERROR: Cannot use abstract type in arithmetic without concrete type
let a: abstract int = 1;
let b = a + 2;  // Error: type mismatch

// CORRECT: Both operands have compatible types
let a: abstract int = 1;
let b: abstract int = 2;

## Overview

This document lists all breaking changes introduced in recent versions of the WGSL parser. Breaking changes are changes that may require modifications to existing code that uses the parser.

## Version 2.0.0

### ❌ Removed: Non-standard Literal Suffixes (US-016)

**Date:** 2025-05-19  
**Epic:** EPIC-004 (Cleanup)  
**Story:** US-016  
**Severity:** **HIGH**

#### Summary

Non-standard literal suffixes `lf`, `li`, and `lu` have been removed to achieve 100% conformance with the WGSL specification. These suffixes were extensions that are not part of the official WGSL CRD.

#### What Changed

| Before | After | Action Required |
|--------|-------|-----------------|
| `42lf` | ❌ Error | Use `42f` instead |
| `42li` | ❌ Error | Use `42i` instead |
| `42lu` | ❌ Error | Use `42u` instead |
| `0x2Alf` | ❌ Error | Use `0x2Af` instead |
| `0x2Ali` | ❌ Error | Use `0x2Ai` instead |
| `0x2Alu` | ❌ Error | Use `0x2Au` instead |

#### Standard WGSL Suffixes

The WGSL specification defines these suffixes:

| Suffix | Type | Description |
|--------|------|-------------|
| (none) | i32 | Signed 32-bit integer |
| `i` | i32 | Signed 32-bit integer (explicit) |
| `u` | u32 | Unsigned 32-bit integer |
| `f` | f32 | 32-bit floating point |
| `h` | f16 | 16-bit floating point (half) |

#### Migration Examples

```wgsl
// BEFORE (Version 1.x)
let longFloat = 3.14lf;    // ❌ No longer works
let longInt = 42li;        // ❌ No longer works  
let longUInt = 42lu;       // ❌ No longer works

// AFTER (Version 2.0+)
let longFloat = 3.14f;    // ✅ Use standard float suffix
let longInt = 42i;        // ✅ Use standard int suffix
let longUInt = 42u;       // ✅ Use standard uint suffix

// Or, if you need a specific type
let floatValue = f32(3.14);  // Explicit type constructor
let intValue = i32(42);      // Explicit type constructor
let uintValue = u32(42);     // Explicit type constructor
```

#### Rationale

The WGSL specification (Section 3.2) only defines `i`, `u`, `f`, and `h` as valid literal suffixes. The `lf`, `li`, and `lu` suffixes were non-standard extensions that caused incompatibility with other WGSL implementations and violated the conformance requirements.

#### Impact Assessment

- **High**: Code using these suffixes will fail to parse
- **Detection**: Search your codebase for `lf`, `li`, `lu` patterns
- **Automated Migration**: Possible with simple find-and-replace

#### Automated Migration Script

```bash

grep -r "[0-9]\+li\b" /path/to/shaders/
grep -r "[0-9]\+lu\b" /path/to/shaders/

# Replace lf with f
find /path/to/shaders/ -type f -name "*.wgsl" -exec sed -i 's/\<[0-9][0-9]*lf\>/\1f/g' {} +

# Replace li with i
find /path/to/shaders/ -type f -name "*.wgsl" -exec sed -i 's/\<[0-9][0-9]*li\>/\1i/g' {} +

# Replace lu with u
find /path/to/shaders/ -type f -name "*.wgsl" -exec sed -i 's/\<[0-9][0-9]*lu\>/\1u/g' {} +
```

// Validate that texture sampling uses correct enumerant types
fun validateTextureSampling(unit: TranslationUnit): Boolean {
    var valid = true
    
    object : ExpressionVisitor<Unit> {
        override fun visitCall(expr: CallExpr) {
            if (expr.function is IdentExpr && expr.function.identifier == "textureSample") {
                // Check that arguments 3 and 4 are AddressMode and FilterMode
                if (expr.args.size >= 5) {
                    val addressModeArg = expr.args[3]
                    val filterModeArg = expr.args[4]
                    
                    if (addressModeArg !is PredeclaredEnumerantExpr || 
                        addressModeArg.category != "AddressMode") {
                        valid = false
                    }
                    
                    if (filterModeArg !is PredeclaredEnumerantExpr || 
                        filterModeArg.category != "FilterMode") {
                        valid = false
                    }
                }
            }
        }
    }.apply { unit.accept(this) }
    
    return valid
}

println("Texture sampling validation: ${validateTextureSampling(unit)}")

### Pretty Printing AST

```kotlin
fun printAst(declaration: GlobalDecl, indent: Int = 0) {
    val prefix = "  ".repeat(indent)
    
    when (declaration) {
        is FunctionDecl -> {
            println("${prefix}Function: ${declaration.name}")
            println("${prefix}  Parameters: ${declaration.parameters.size}")
            println("${prefix}  Return type: ${declaration.returnType}")
            declaration.body?.let { body ->
                println("${prefix}  Body:")
                body.statements.forEach { stmt ->
                    println("${prefix}    Statement: $stmt")
                }
            }
        }
        is GlobalVariableDecl -> {
            println("${prefix}Variable: ${declaration.name}")
            println("${prefix}  Type: ${declaration.type}")
            println("${prefix}  Expression: ${declaration.expr}")
        }
        is StructDecl -> {
            println("${prefix}Struct: ${declaration.name}")
            declaration.members.forEach { member ->
                println("${prefix}  Member: ${member.name}")
                println("${prefix}    Type: ${member.type}")
            }
        }
        is EnumDecl -> {
            println("${prefix}Enum: ${declaration.name}")
            declaration.members.forEach { member ->
                println("${prefix}  Member: ${member.name}")
                println("${prefix}    Value: ${member.value}")
            }
        }
    }
}

## Performance Tips

### Incremental Parsing

For large codebases, consider parsing files incrementally:

```kotlin
fun parseShaderFile(file: File): Result<TranslationUnit> {
    return try {
        val source = file.readText()
        val parser = Parser(Lexer(source))
        val unit = parser.parseAndValidate()
        
        if (parser.errors.isNotEmpty()) {
            Result.failure(ParseException(parser.errors))
        } else {
            Result.success(unit)
        }
    } catch (e: Exception) {
        Result.failure(e)
    }
}

// Parse all shaders in a directory
fun parseShaderDirectory(directory: File): Map<File, Result<TranslationUnit>> {
    return directory.listFiles { it.extension == "wgsl" }
        ?.associateWith { file -> parseShaderFile(file) }
        ?: emptyMap()
}
```

### Caching Parsed Results

```kotlin
class ShaderCache {
    private val cache = mutableMapOf<String, Result<TranslationUnit>>()
    
    fun getOrParse(source: String): Result<TranslationUnit> {
        return cache.getOrPut(source) {
            try {
                val parser = Parser(Lexer(source))
                val unit = parser.parseAndValidate()
                
                if (parser.errors.isNotEmpty()) {
                    Result.failure(ParseException(parser.errors))
                } else {
                    Result.success(unit)
                }
            } catch (e: Exception) {
                Result.failure(e)
            }
        }
    }
    
    fun clear() = cache.clear()
}

// Usage
val cache = ShaderCache()
val result1 = cache.getOrParse(shader1)
val result2 = cache.getOrParse(shader2)  // Uses cache if same as shader1
```
