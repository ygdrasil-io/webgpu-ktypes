## 📋 OBJECTIFS

Implémenter le backend **Metal Shading Language** (MSL) qui transforme l'IR Naga en code source MSL pour les plateformes Apple (macOS, iOS, tvOS, watchOS).

**Livrable** : Module `naga-msl` capable de générer du code MSL valide.

## 🎯 CONCEPTS CLÉS

### 1. MSL en bref
- **Langage** : Metal Shading Language (Apple)
- **Versions** : 1.0 (2015) → 2.3 (2022, actuelle)
- **Plateformes** : macOS 10.11+, iOS 8+, tvOS, watchOS
- **Version par défaut** : MSL 2.3

### 2. Différences clés MSL/WGSL

| Feature | WGSL | MSL |
|---------|------|-----|
| Binding model | Hierarchique (group+binding) | Plat (index unique) |
| Entry point varyings | Arguments separés | Struct avec `[[stage_in]]`/`[[stage_out]]` |
| Noms de types | `texture_2d<f32>` | `texture2d<float>` |
| Matrices | Column-major | Row-major par défaut |
| External textures | Natif | Lowered vers 3 textures + buffer params |
| Pointers | Limités | Support natif |

### 3. Binding Model MSL

WGSL:
```wgsl
@group(0) @binding(0)
var tex: texture_2d<f32>;
@group(0) @binding(1) 
var sam: sampler;
```

MSL (avec BindingMap):
```msl
texture2d<float> tex [[texture(0)]];
sampler sam [[sampler(1)]];
```

### 4. Entry Points

Les inputs/outputs sont regroupés dans des structs:
```msl
struct VertexInput {
    float4 position [[location(0)]];
};

struct VertexOutput {
    float4 position [[location(0)]];
};

vertex VertexOutput vertexMain(
    device VertexInput input [[stage_in]]
) {
    VertexOutput output;
    output.position = input.position;
    return output;
}
```

### 5. External Textures

Une `texture_external` est lowered vers:
- 3 `texture2d<float>` (planes Y, Cb, Cr)
- 1 `constant NagaExternalTextureParams` (métadonnées)

Fonctions helpers générées pour `textureDimensions()`, `textureLoad()`, `textureSampleBaseClampToEdge()`.

### 6. OOB Locals

Pour `BoundsCheckPolicy::ReadZeroSkipWrite` sur les pointeurs:
- Création de variables dummy (une par type par fonction)
- Utilisées quand l'index est out-of-bounds

## 🎯 IMPLÉMENTATION

### 1. MslOptions.kt

Options spécifiques MSL avec:
- `bindingMap: BindingMap` - Mapping WGSL→MSL
- `inlineSamplers: Boolean` - Samplers inline
- `bufferSizeAlignment: Int` - Alignement buffers
- Version par défaut: "2.3"

### 2. BindTarget.kt

```kotlin
data class BindTarget(
    val buffer: Slot?,      // [[buffer(N)]]
    val texture: Slot?,     // [[texture(N)]]
    val sampler: BindSamplerTarget?,  // [[sampler(N)]] ou [[sampler(inline, N)]]
    val externalTexture: BindExternalTextureTarget?,
    val mutable: Boolean
)

sealed class BindSamplerTarget {
    data class Resource(val slot: Slot)
    data class Inline(val index: InlineSamplerIndex)
}

data class BindExternalTextureTarget(
    val planes: List<Slot>,  // 3 planes max
    val params: Slot
)
```

### 3. MslWriter.kt (Classe principale)

Hérite de `WriterBase<MslOptions>` et implémente:

**Header:**
```kotlin
// Metal Shading Language
// Generated by WebGPU-KTypes
#include <metal_stdlib>
#include <simd/simd.h>
```

**Types:**
- Structs → `struct Name { ... };` avec attributs `[[align(N)]]`
- Vecteurs → `float2`, `float3`, `float4`, `int2`, etc.
- Matrices → `float2x2`, `float3x3`, `float4x4`
- Images → `texture2d<float>`, `texture3d<float>`, etc.

**Expressions:**
- Littéraux: `true`, `false`, `42`, `42u`, `3.14`
- Identifiants: noms générés par Namer
- Opérateurs: `+`, `-`, `*`, `/`, `%`, `==`, `!=`, `<`, `>`, `<=`, `>=`, `&&`, `||`, `&`, `|`, `^`, `~`, `<<`, `>>`
- Select: `select(accept, reject, condition)`
- Compose: `float4(a, b, c, d)`
- Splat: `float4(value)`
- Swizzle: `vec.xyzw`
- Access: `arr[index]`
- AccessIndex: `arr[0]`
- Cast: `float(value)` ou `intBitsToFloat(value)`
- Math: fonctions built-in MSL

**Statements:**
- Bloc: `{ ... }`
- If/Else: `if (cond) { ... } else { ... }`
- Switch: `switch (value) { case 1: ... break; default: ... }`
- Loop: `while (true) { ... if (cond) break; }`
- While: `while (cond) { ... }`
- For: `for (init; cond; update) { ... }`
- Break/Continue: `break;`, `continue;`
- Return: `return value;` ou `return;`
- Discard: `discard_fragment();`
- Store: `ptr = value;`
- Atomic: `atomic_fetch_add(ptr, value);`

**Entry Points:**
- Détecter les inputs/outputs
- Générer les structs `Name_input` et `Name_output`
- Assigner les arguments depuis les structs
- Gérer les globals non-Private comme arguments

**External Textures:**
- Détecter les `ImageClass::EXTERNAL`
- Générer les 3 textures + buffer params
- Générer les fonctions helpers

## 📝 NOTES

1. **Priorité** : MSL est le backend principal pour les plateformes Apple. Il doit être complet et robuste.

2. **BindingMap obligatoire** : Sans un BindingMap correct, les bindings ne seront pas traduits correctement.

3. **External Textures** : C'est la partie la plus complexe. Prévoir du temps pour les tests.

4. **Versions MSL** : Supporter MSL 2.0+ par défaut, avec option pour les versions antérieures si nécessaire.

5. **Performance** : Utiliser StringBuilder pour éviter les allocations inutiles.

6. **Validation** : Valider que le module utilise uniquement des fonctionnalités supportées par MSL.
